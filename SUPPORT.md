## Public Support Paths

- documentation gaps and usability issues: GitHub issue in the future public repo
- model and publishing questions: attach the release bundle and validation status
- validation or pilot questions: use the private pilot runbook and include all manifests
- security issues: do not use public issues; follow [SECURITY.md](SECURITY.md)

## What to Include

- TuneForge version or git SHA
- hardware and OS
- provider or backend used
- config path
- exact command
- log excerpt or `protocol.jsonl` sample
- whether the issue affects preview claims, publishing, or validation evidence

TuneForge support is engineering support. It is not legal advice and it does not replace Austrian or EU legal counsel.
