# TuneForge Compliance Pack

## Purpose

TuneForge is designed to help teams build locally governed fine-tuning workflows. The public compliance pack is documentation support for engineering and governance, not legal advice.

## What the Pack Includes

- model card template
- training manifest schema
- benchmark summary schema
- release checklist
- provenance and license guidance
- intended-use and limitation guidance
- DPIA and VVT input framing for internal governance
- EU AI Act positioning notes for low-risk documentation workflows

## What the Pack Does Not Claim

- it does not certify legal compliance
- it does not replace a lawyer or data protection officer
- it does not make a base model lawful for every dataset or deployment

## Austrian and EU Positioning

The documentation is written for teams that need:

- DSGVO-aware handling of datasets and model releases
- internal documentation for Austrian and EU governance reviews
- traceable publication of benchmarked adapters and GGUF artifacts

## Minimum Public Evidence

Every public TuneForge model release should expose:

- data provenance summary
- base model and derivative license summary
- intended use
- out-of-scope use
- limitations
- benchmark summary
- hardware context
- publication date and version
